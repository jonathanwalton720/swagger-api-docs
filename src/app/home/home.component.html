Authentication

Each request must be authenticated with your API key. Please contact us to get your API key. Only you should have access to your API key.

Include the header parameter x-api-key and use your API key to authenticate each request.

The Version endpoint can be used to test authentication. The example includes all necessary header information, with the x-api-key header in addition to your standard Accept and Host headers. It will fail if the API key header is missing or not in the correct format.

Possible Responses
Status Code Returns
200 - OK "v1"
401 - Unauthorized "Key not authorized to access this resource."
403 - Forbidden "Key not found in header or not correct format."
Example Request
GET {{apiEndpoint}}/v1/Version HTTP/1.1
User-Agent: Fiddler
Host: {{apiEndpoint.replace(httpRegExp, '')}}
Accept: application/json
x-api-key: b2e7ac1f-3efe-4a64-99db-14ebae93ae2b

Paging

The API will return paged results when requesting a list of objects. You may define the page and pageSize parameters when creating a request.

The first page will always be 1. The response will include an x-total-count header with an integer value to determine how many total items matched the request parameters.

The API will only return the number of resources equal to or fewer than the pageSize. The default pageSize is 10.
